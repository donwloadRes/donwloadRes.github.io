# Python+Django+Vue 后台管理系统

欢迎使用基于Python的Django框架结合前端Vue.js技术栈开发的后台管理系统。本系统旨在提供一套简洁高效的用户及用户组管理解决方案。适合需要构建快速原型或小型企业管理系统的开发者。

## 特性概览

- **用户管理**：实现用户的增删改查操作，全面控制用户权限。
- **用户组管理**：细粒度的用户组设置，方便进行权限分配和管理。
  
## 技术栈

- **后端**: Django (Python)
- **前端**: Vue.js + Element UI（或Vuetify）
- **数据库**: MySQL
- **版本控制**: Git

## 系统要求

- Python 3.6 或更高版本
- Django 2.x/3.x（请根据项目具体版本选择）
- Node.js (用于Vue.js的开发与构建)
- npm 或 yarn（作为Node.js的包管理器）

## 快速入门

1. **环境准备**：确保你的开发环境中安装了Python、pip、Node.js以及npm/yarn。
2. **克隆项目**：
   ```bash
   git clone https://github.com/your-repo-url.git
   ```
   替换`https://github.com/your-repo-url.git`为你实际的项目仓库地址。
   
3. **安装依赖**：
   - 在项目根目录下运行以下命令安装Python依赖：
     ```bash
     pip install -r requirements.txt
     ```
   - 进入前端目录并安装JavaScript依赖：
     ```bash
     cd frontend
     npm install
     ```
4. **数据库设置**：
   - 将`mysql数据库表的信息user_group.sql`导入到你的MySQL数据库中。
   - 配置Django项目的数据库连接详情（位于`settings.py`）。

5. **运行项目**：
   - 先迁移数据库结构：
     ```bash
     python manage.py migrate
     ```
   - 创建超级用户（如果需要进行管理界面登录）：
     ```bash
     python manage.py createsuperuser
     ```
   - 运行服务器：
     ```bash
     python manage.py runserver
     ```
   - 打开浏览器访问 http://localhost:8000 即可查看后端服务状态；前端应用需根据项目配置启动。

6. **前端开发服务器**：
   （仅在开发阶段）在前端目录下启动Vue开发服务器：
   ```bash
   npm run serve
   ```
   这将在另一个端口上启动Vue应用，通常为`http://localhost:8080`。

## 注意事项

- 请确保在处理用户数据时遵循相关隐私政策和法律法规。
- 根据实际需求调整数据库模型和前后端接口。
- 开发过程中请注意代码安全，避免SQL注入等常见安全漏洞。
