# C++ CMake 多源文件编译指南

## 资源文件描述

本资源文件提供了一个使用 `CMakeLists.txt` 编译 C++ 项目的方法，适用于同目录下存在多个源文件的情况。通过本指南，您将学会如何配置 `CMakeLists.txt` 文件，以便在同一目录下编译多个源文件并生成可执行文件。

## 内容概述

本资源文件包含以下内容：

1. **CMakeLists.txt 配置文件**：详细说明了如何在 `CMakeLists.txt` 中配置多个源文件，以便生成可执行文件。
2. **示例代码**：提供了一些简单的 C++ 源文件示例，帮助您理解如何将多个源文件组织在一起进行编译。
3. **编译步骤**：详细介绍了使用 CMake 进行编译的步骤，包括如何生成构建文件、编译项目以及运行生成的可执行文件。

## 使用方法

1. **下载资源文件**：将本资源文件下载到您的本地目录。
2. **配置 CMakeLists.txt**：根据您的项目需求，修改 `CMakeLists.txt` 文件中的源文件列表。
3. **生成构建文件**：在项目根目录下运行 `cmake .` 命令，生成构建文件。
4. **编译项目**：运行 `make` 命令，编译项目并生成可执行文件。
5. **运行可执行文件**：在终端中运行生成的可执行文件，验证编译结果。

## 注意事项

- 确保您的系统中已安装 CMake 和编译工具链（如 GCC 或 Clang）。
- 在修改 `CMakeLists.txt` 文件时，注意源文件的路径和名称，确保它们与实际文件一致。
- 如果您在编译过程中遇到问题，请参考 CMake 的官方文档或相关社区资源进行排查。

## 总结

通过本资源文件，您将掌握使用 CMake 编译 C++ 项目的基本方法，特别是如何处理同目录下的多个源文件。希望本指南能够帮助您顺利完成项目的编译工作。
